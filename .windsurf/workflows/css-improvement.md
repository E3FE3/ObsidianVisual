# Workflow: Улучшение CSS Dashboard

**Триггер**: `/css-improve`

**Назначение**: Анализ и улучшение CSS стилей Obsidian dashboard для современного, профессионального внешнего вида

## Шаги

1. **Прочитать текущий CSS**
   - Прочитать `.obsidian/snippets/dashboard.css`
   - Проанализировать текущую реализацию
   - Определить области для улучшения

2. **Прочитать Dashboard HTML**
   - Прочитать `_Dashboard/Home.md`
   - Проверить использование CSS классов
   - Проверить grid структуру

3. **Проанализировать дизайн паттерны**
   - Проверить консистентность цветов
   - Проверить spacing и типографику
   - Оценить hover эффекты и анимации
   - Проверить responsive breakpoints

4. **Предложить улучшения**
   - Перечислить конкретные улучшения
   - Объяснить дизайн rationale
   - Показать сравнение до/после
   - Оценить визуальное влияние

5. **Реализовать изменения**
   - Обновить dashboard.css с улучшениями
   - Сохранить существующую структуру классов
   - Добавить новые фичи если нужно
   - Обеспечить обратную совместимость

6. **Синхронизировать файлы**
   - Скопировать обновлённый CSS в Windows
   - Команда: `cp .obsidian/snippets/dashboard.css /mnt/c/Users/E3FE3/Folder_F/Projects/Obsidian/Vaults/LAUNCHER/.obsidian/snippets/dashboard.css`

7. **Создать тестовый файл** (опционально)
   - Сгенерировать test.md со всеми компонентами
   - Показать визуальные примеры изменений
   - Задокументировать новые классы если добавлены

## Ожидаемый результат

- Обновлённый файл dashboard.css
- Список сделанных улучшений
- Визуальное сравнение или описание
- Инструкции для тестирования в Obsidian

## Заметки

- Фокус на современной эстетике веб-приложений (стиль Linear, Vercel, Notion)
- Обеспечить mobile responsiveness
- Поддерживать русский язык в описаниях
- Оставлять технические термины на английском
