# Workflow: Пакетный импорт команд

**Триггер**: `/batch-import`

**Назначение**: Автоматическое создание файлов команд (Snippets) на основе существующих скриптов в папке `Scripts/` или внешних файлов.

## Шаги

1. **Поиск скриптов**
   - Просканировать папку `Scripts/`
   - Найти все `.sh`, `.ps1`, или `.py` файлы
   - Вывести список найденных файлов для подтверждения

2. **Извлечение метаданных**
   - Для каждого скрипта попытаться определить:
     - `system` (по путям или комментариям)
     - `technology` (по расширению или ключевым словам)
     - `description` (из первых строк комментариев)
     - `tags` (на основе технологии и системы)

3. **Генерация файлов команд**
   - Для каждого скрипта создать файл в `Snippets/`
   - Использовать шаблон `_Templates/Command.md`
   - Заполнить frontmatter данными из шага 2
   - В тело файла добавить:
     - Описание
     - Блок кода с вызовом скрипта или его содержимым
     - Примеры использования

4. **Обновление Dashboard**
   - Проверить, что новые команды отображаются в `_Dashboard/All Commands.md`
   - Обновить счетчики в `Home.md` (если используются статические данные)

5. **Завершение**
   - Вывести отчет о создании команд
   - Предложить закоммитить изменения в Git

## Пример шаблона импорта

```markdown
---
system: {{system}}
technology: {{technology}}
tags: [{{tags}}]
created: {{date}}
description: {{description}}
---

# {{title}}

## Описание
{{description}}

## Команда
```bash
./Scripts/{{filename}}
```
```

## Заметки
- Использовать этот workflow при добавлении новых инструментов в инфраструктуру
- Помогает поддерживать документацию в актуальном состоянии вместе с кодом
- Обеспечивает консистентность метаданных во всем vault
