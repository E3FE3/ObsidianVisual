# Workflow: Создание новой команды

**Триггер**: `/new-command`

**Назначение**: Создать новый файл документации команды для LAUNCHER vault

## Шаги

1. **Собрать информацию**
   Спросить у пользователя:
   - Название команды
   - Система (WINDOWS, WSL, TheYvnHub, TheYvnAI)
   - Технология (rclone, ssh, podman, git, и т.д.)
   - Описание команды (на русском)
   - Саму команду или скрипт
   - Примеры использования

2. **Прочитать шаблон**
   - Прочитать `_Templates/Command.md`
   - Использовать как базовую структуру

3. **Создать файл**
   - Определить расположение: `Snippets/` или `Scripts/` в зависимости от сложности
   - Имя файла: описательное английское название с пробелами (например, "SSH TheYvnHub.md")

4. **Заполнить Frontmatter**
   ```yaml
   ---
   system: [из input]
   technology: [из input]
   tags: [авто-генерировать релевантные теги]
   created: [сегодняшняя дата YYYY-MM-DD]
   description: [из input, на русском]
   ---
   ```

5. **Структурировать контент**
   - # Заголовок (с emoji)
   - ## Описание (описание на русском)
   - ## Команда (команда в code block)
   - ## Примеры использования (примеры с вариациями)
   - ## Заметки (примечания, требования, предупреждения)
   - ## Связанные (ссылки на родственные команды)

6. **Валидация**
   - Проверить синтаксис frontmatter
   - Проверить что code blocks имеют language tags
   - Убедиться что описания на русском
   - Добавить релевантные Obsidian ссылки

7. **Обновить индекс**
   - Команда авто-появится в `All Commands.md` через Dataview
   - Проверить что появляется на релевантных страницах систем/технологий

## Ожидаемый результат

- Новый файл команды создан в подходящем расположении
- Правильно отформатирован с frontmatter
- Включает примеры и заметки
- Связан с родственными командами

## Заметки

- По умолчанию использовать `Snippets/` для простых команд
- Использовать `Scripts/` для многострочных скриптов или сложной автоматизации
- Всегда включать хотя бы один пример использования
- Предупреждать о любых необходимых зависимостях
